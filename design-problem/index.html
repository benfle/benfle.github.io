Design Problem

#

In Notes on the Synthesis of Form, Christopher Alexander represents a design problem as a set of misfits and the coupling between those.

#

In the Axiomatic Design method, a design problem is represented as a tree of independent functional requirements and constraints.

#

The analysis of the problem is the most important task in the design process. We cannot design a good solution if we do not have a clear definition of the problem.

#

In particular, it is important to distinguish the problem domain from the solution domain. The problem must be described as functional requirements in the problem domain. However, we usually have difficulty thinking in terms of function. We always have a tendency to leap to a solution. I illustrate this problem in software engineering teams in "The Problem with User Stories".

#

The designer is not going to find the right problem to solve the first time. The problem might need to be reformulated several times during the design process.

#

Moreover, in the top-down processes like the Axiomatic Design method, the designer needs to design the solution of a level before deciding what the functional requirements at the lower level are. So there is no real problem definition phase per se but a back and forth between the tree of functional requirements and the tree of design parameters.

Conceptual Design of Software

#

A conceptual design is a description of a software as a network of concepts. Those concepts are chosen, or invented, to solve the user's problem.

#

This design does not describe how the software is going to be represented in the machine or how it is going to look like to the user. This design is focused on how the software works.

#

In the software industry, we have a tendency to overlook this level of description and jump to solutions. "The Problem with User Stories", illustrates some of these problems.

#

The Software Design Group at MIT is working on the conceptual design of software. The best introduction to their work is probably the article by Daniel Jackson "Towards a Theory of Conceptual Design for Software."

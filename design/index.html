Design

#

A design is a description of the structure of an artifact as a set of design parameters and the coupling between those. A design parameter is a unit of a design. It can be qualitative or quantitative.

#

To design an artifact is to map a set of requirements from the problem domain to a set of design parameters in the solution domain.

#

#

There are many ways to represent a design problem and many ways to identify the solution. Designers usually go back and forth between the problem and the solution until they're satisfied with their design.

#

The interactions between the requirements, or coupling, is what makes the problem hard to solve.

#

In Notes on the Synthesis of Form, Christopher Alexander presents a method to decompose the design problem into independent clusters of misfits. Once those clusters are identified, each can be solved independently. Those individual solutions are called constructive diagrams, what we now call Design Patterns.

#

In the Axiomatic Design method, the designer is encouraged to identify a minimal set of independent functional requirements and to design a solution in a way that keeps the requirements independent.

#

In particular, the designer must be careful not to couple requirements when designing a solution. The designer should be able to tweak a design parameter to adjust to a requirement without impacting other requirements. "Decoupling Requirements" illustrates this problem.
